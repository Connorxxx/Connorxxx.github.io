Hexo+Github+VPS 同步方案

在 GitHub 部署完 Hexo, 使用了之后非常喜欢, 博主原来的博客是用 WordPress 搭建的, 总所周知的原因在 WordPress 上使用 MarkDown 写博客十分的痛苦, 就算是使用插件也有从写到发布的烦恼, 所以干脆就把原来的 WordPress 换成 Hexo, Hexo 可以直接用 MarkDown 写, 写完后一条命令就能发布, 可以说是非常的方便和畅快了.

博客搭建

前提

请确保 VPS 安装了下列程序:

  • Nginx
  • Git

这里假定已经安装完了, 我就不赘述一般安装过程了.

开始搭建

在 Nginx 的站点上 clone 下在 Github 上托管的源码.进入站点所在的目录, 我的是/www/wwwroot/blog.wuyinghao.cn, 然后把 GitHub 上的源码 clone 下来, 并且修改配置文件 server 下的 root 字段值修改为 clone 后的博客目录路径

我的博客在 GitHub 的仓库名是 Takuwy.github.io, 所以修改为:

root /www/wwwroot/blog.wuyinghao.cn/Takuwy.github.io;

现在可以通过域名访问存放在 VPS 上的博客了, 但还没完, 如果我们写了新的文章执行 hexo g -d 时只会将文章同步到 GitHub 上 而在VPS上则没有同步.

同步

创建 git 仓库

想要在 Github 上和 VPS 都同步文章, 首先创建 git 用户:

adduser git

在 /home/git 下创建 hexo.git 空仓：

cd /home/git

git init --bare hexo.git

修改 hexo.git 的所有者和组权限：

chown -R git:git hexo.git

获取本地电脑 SSH Key：

cat ~/.ssh/id_rsa.pub

将 SSH Key 复制到 VPS 上的 /home/git/.ssh/authorized_keys 文件中.

没有文件就创建给权限, 别问我给多大的权限, 问就是 777! (往下的操作都是没有文件的就创建)
